互联网的遗留问题

· 2020-10-12 · # Failure # Technology & Society

上周，我听了一个在线演讲《去中心化互联网的理论与实践》，意外发现这和我想要解决的问题高度一致。我在过去的思考中发现：

  • 如果承认好的内容需要付费，而免费的内容一定会被更多人消费的先决条件，那么这天然在促进内容的劣币驱逐良币。（但这并不是必然的）
  • 中心化产品服务大多数人的需求，但每个人在特定场景下都可能是小众人群。
  • 今天的互联网并没有促进产品间的协同能力（甚至是闭塞，回忆一下微信屏蔽了多少产品的分享链接）。

这也是我尤其关注开源、利基、维基百科的原因。近期 Scriptable 的流行是一个很好的例子，这个 App 允许用户调用接口，编写程序，并通过 iOS 14 的小组件呈现个性化内容：

  • 每个人都有特定的需求，这些需求可能是非常小众的（例如，有人在小组件上显示 GitHub Contributions、显示当天所剩小时数、显示微博热搜等等）。
  • 代码是可复用的，一个人编写的代码可以被无数人使用，其他人只需要修改个性化参数即可。
  • 实际上，它增强了软件间的协同能力，现在用户已经可以在一个屏幕里看到多个自己真正需要的信息。（让人想到 the api economy）

这一变化是巨大的，此前，用户只能被动使用已经存在的工具，而现在则能创造自己真正需要的。但这个过程仍然存在一些问题——代码的门槛还是过高了。尽管 Javascript 已经是一门相对易学的语言了（它的作者 Brendan Eich 只花了 10 天时间就编写出这门语言），但过去的经验告诉我们，这个世界并没有朝着“人人都是开发者”的方向发展，而是一部分开发者创造出 NoCode/LowCode 工具，让其他人低成本使用。

实际上，这也正是 Notion 的内核，和印象笔记等工具产品不同，Notion 不要求用户顺从产品开发者设定的规则，允许用户通过可视化的方式（NoCode），实现自己的独特需求，被创建的页面同样可以作为模板被分享给别人（在国外，有人创建垂类市场的模版，并以此盈利）。
