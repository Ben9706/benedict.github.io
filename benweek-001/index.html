打造移动端的好莱坞，Quibi 忽视了什么？

这是我参与的一篇文章。今年 4 月，卡森伯格的新创业项目 Quibi 正式上线（此前卡森伯格和斯皮尔伯格合办梦工厂）。作为一手打造了迪士尼动画部门的传奇人物，卡森伯格一直是好莱坞的风云人物，Quibi 也被寄予厚望。Quibi 试图做一个高质量的短视频产品，但目前从市场表现看非常糟糕。本文梳理了卡森伯格在今年 1 月 CES 上的演讲，介绍他为什么认为 Quibi 的模式能够成立，且基于逻辑分析他的思考里可能缺漏了什么。


Markdown 转微信公众号排版样式

我非常讨厌微信公众号的排版后台，想知道它有多难用，只要看秀米、135编辑器等多么热门就知道了（考虑到今年微信公开课 Always Beta 上，张小龙宣称公众号原本的定位并不是长图文，这种难用突然变得能够理解了）。为了避免排版带来的繁琐工作，我使用并修改了开源程序 wechat-format，以符合自己的排版习惯，实现 Markdown 转公众号富文本的功能，并通过 Github Page 供其他朋友一起使用。如果你也有类似的需求，可以尝试该方法。

美国考虑封禁 TikTok

2018 年 Q3，TikTok 在印度尼西亚因为内容违规被封禁一周，此后，围绕 TikTok 的纷争越来越多，继印度下架中国 59 款 App 后，美国也考虑以安全隐私为由封禁 TikTok。表面上看，核心的冲突点是「数据」，美国认为 TikTok 的母公司字节跳动可能将美国用户数据传回中国（目前尚无证据证明，所有的说辞都添加上未来可能这一说法）。这非常奇怪，因为这种冲突理论上完全可以从技术角度去规避，尤其考虑到字节跳动在 2019 年初就开始将中国与海外进行访问权限的分割，已经做了非常多全球化努力。

华盛顿邮报的这篇文章从技术的角度认为，首先 TikTok 没有比 Facebook 获取更多的个人信息，其次，Tiktok 宣称所有美国用户的数据都存储在 AWS 上，那么完全可以管控数据流出方向（同样也合理），最后，所有当下从技术层面的分析都没有发现数据有被 TikTok 滥用的情况。

在字节跳动能在海外建立诚信吗？中，作者认为建立信任的最佳方式是开源。“Talk is cheap. Show me the code.”此外，Ben Thompson 在 The TikTok War 中也谈及很多地缘政治话题。


谷歌前高管试图推出订阅制搜索服务

Google 前广告部门高管 Sridhar Ramaswamy 对于过度依赖广告的商业模式感到厌恶，他认为在这一过程中，用户牺牲了太多的隐私和体验，原本就已经信息过载了，用户还不得不被迫接受无止尽的广告。因此，他开办了 Neeva，希望在用户搜索高效的同时，保护用户的隐私。

Neeva 并不是一个全新的搜索引擎，更像一个整合工具：搜索排名由微软必应提供，天气信息来自天气网站，股票数据来自 Intrinio，地图则来自苹果。当用户登录 Google、微软或是 Dropbox 账户后，Neeva 也可以在个人文件中进行搜索。Ramaswamy 打算采用订阅制的方式推出该服务，暂计划用户可以免费使用到今年年底，然后每月收取 10 美元不到的订阅费。

搜索引擎适合订阅制吗？这个问题没有具体答案，我想举两个例子。

  • 电视节目。在 HBO 以前，电视一直依赖广告盈利，而 HBO 带来订阅电视的成功案例。
  • 天气预报。天气预报是一个长期更新的，且一定程度上刚需的服务，但天气预报并未发展成订阅制服务（想一下墨迹天气上市失败的案例）。

出于长期对搜索结果的不满，我对这个产品很感兴趣。目前 Neeva 仍未披露更多信息，你可以阅读 Neeva 的博客，了解他们做该服务的愿景。


亲历者讲述 Twitter 遭遇黑客攻击事件

几名参与 Twitter 黑客攻击事件的亲历者讲述这段逐渐失控的故事。发动此次攻击的并不是某个经验丰富的黑客组织，相反，它是由一群年轻人完成。最初，这群年轻人只是想出售一些特殊的 Twitter 号，例如一个 @y 的 Twitter 名被人以 1500 美元的价格拍卖下，很多人觉得这种名字特别酷，但随后这件事开始变得失控。

还有 100 多天就要进行美国总统选举，如果黑客以拜登的账号发布信息，宣布退出美国总统竞争，亦或是在选举开始之后，这名黑客发起攻击。这些都可能造成更严重的政治问题，而这些原本都有可能发生。

和这些比较起来，以名人的口吻发布一些非常容易甄别的信息，就像是一场恶作剧。

交通工具面料选择

这篇文章提到如何设计交通工具上座椅的面料材质，这种材质需要满足几个特点：

  • 耐用。每天有很多人使用，它必须能够使用很多年。
  • 耐脏。它需要易于清洁，而且看上去不太脏。
  • 安全。交通工具可能位于一些极端环境，必须考虑到是否易燃等。

生活中这类细节实在太多了。上周我前往一个新营业的咖啡厅，它在杯面上的贴纸是遇到水后会变色的类型，而夏天很多人会选择点冰咖啡，这使得当天在社交媒体上传播的照片贴纸大多都一块深一块浅的，显得廉价。


🚀 观点

无障碍访问性

一份《信息技术 互联网内容无障碍可访问性 技术要求与测试方法》的国家标准在今年 3 月起实施，这份文件旨在提升残障人群使用数字产品的体验。尽管这份文件是 GB/T，即推荐性而非强制性国标，但对于无障碍立法仍有巨大意义。

回顾无障碍的历史，早期解决无障碍的方式是特殊问题特殊处理，例如发明盲文和手语。但由于这种处理方式有较高的学习成本，沟通形式异于大众，一定程度上加剧了残障人士脱节于普通大众。（但需要说明，盲文最早可追溯到 1786 年的法国，并在 1821 年由一名盲人路易·布莱叶完善，而第一台计算机出现于 1946 年，盲文和手语的出现有历史必然性）

那么数字产品是如何改进这一问题的？在手机上，正常用户和残障人士使用的是同一种 UI 界面，仅在交互方式上有不同，在信息的获取和交流上没有差异。（想要了解更多有关无障碍的话题，可以阅读这本书《信息无障碍：提升用户体验的另一种视角》。）

我认为这背后传递的观点是，不要有意识地去孤立群体，而应该给予更平等的环境。

此外这让我想到另一组数据，美国有 69% 的儿童使用 Youtube（Youtube 同样也是儿童使用最多的应用），但只有 7% 的儿童使用 Youtube 儿童版。

另一方面，对儿童（或其他群体）的保护又是切实需要的，如何找到这种平衡点？思考至此，这个话题又延伸至家长式立法（Paternalism）的辩论上。


盗号灰产和数字遗产

这个例子里，我认为核心不是阻止盗号灰产，而是面对更多数字时代下的死亡，我们如何处理数字遗产。现实社会里，通过空间分隔出墓地和生活空间，而网络上的亡者和生者却共处在同一个社交媒体上，这种特性使得李医生的微博成为一面哭墙，也使得平凡人的悼念变得唾手可及。

Facebook 的做法是，在自己过世以前，可以指定一位联系人管理自己去世后的悼念帐户，或者也可以选择从 Facebook 永久删除帐户。悼念帐户的作用即提供一个场所供亲朋好友缅怀追思，而且在用户过世后，该账号不会再作为「可能认识的人」被推荐给其他用户。

如果想了解更多科技公司的应对措施，可以阅读这篇文章。
